# Quarterly OKR Drafting

Every quarter I need to draft OKRs (Objectives and Key Results) and review them with the team. The current process is: I think about what I want to accomplish, write 3-5 objectives, debate with myself, ship.

Want a workflow that:
1. Pulls the previous quarter's OKRs from my OKR note (in the vault)
2. Pulls all atomic notes tagged with the previous quarter's themes
3. Identifies the 3-5 themes that emerged in the work
4. Drafts new OKRs that build on those themes
5. Each OKR has 3-5 key results that are *measurable* (numbers, dates, or specific outcomes)
6. Outputs a markdown document with: previous quarter review, new OKRs, dependencies, risks

Tone: should be ambitious but achievable. Each KR should be falsifiable.

This is more strategic than the investor update — it requires synthesizing the previous quarter's work. The Wholeness-Engine could grade the draft (does each KR have a clear success criterion? is the level of ambition right?).
