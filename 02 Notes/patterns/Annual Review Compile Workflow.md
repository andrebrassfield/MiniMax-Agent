# Annual Review Compile

End of year. I need a comprehensive annual review document: what I accomplished, what I learned, what I want to do differently.

Inputs:
- All 12 monthly briefs (from `01 Daily/2026-MM-DD.md` and `06 Connections/`)
- All atomic notes from the year (CHIEF-format, in 02 Notes/)
- All git commits (from the vault + side projects)
- A few photo albums (Dropbox, iCloud)
- A handful of "highlights" I want to emphasize (I'll list these manually)

Process:
1. Pull everything into a year-end synthesis
2. Cluster notes by theme (probably 5-8 themes emerge)
3. For each theme, write a 2-3 paragraph summary citing the key notes
4. Add a "what I learned" section (the meta-lessons)
5. Add a "what I'll do differently" section (the commitments)
6. Generate a 1-page TL;DR
7. Add a timeline visualization (Gantt-style or just a markdown table)
8. Send to me for review
9. After approval, share with a small circle (5-10 people)

This is a once-a-year workflow. It doesn't need to be fast, but it needs to be *comprehensive* — missing a major accomplishment would be embarrassing.

Storage: 04 Resources/Annual-Reviews/2026.md

PatternForge: yes. Skill Pack: yes (the skill would do the data pull + theme clustering).
