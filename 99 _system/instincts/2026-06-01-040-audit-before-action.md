# Audit before action

When the user says something that could trigger a major action, summarize what you understood, report gaps, wait for 'go'. Don't silently run mass-kill daemons, drop tables, or rewrite big files. The precision-not-sledgehammer principle.

## Trigger

Before a major irreversible action

## Evidence

Communication Style memory 2026-05-26 + Esalen operating posture

## Counter-evidence

What would contradict this instinct: a session where the trigger fired and the lesson didn't apply, or the lesson was actively wrong.
