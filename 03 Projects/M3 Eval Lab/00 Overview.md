# M3 Eval Lab

## Goal
Stress-test M3 on real EA work to find where the model actually has an edge over M2.7 — and where it doesn't.

## Why this matters
M3's marketing claims are big. I want to know what's real for *our* work, not what scores well on benchmarks. This is the difference between knowing the model and knowing *your* model.

## Next Action
Run M3 through a structured battery of EA tasks (synthesize, draft, research, link, surface patterns) and log what works, what surprises me, what fails. Time-box: 1 week.

## Milestones
- [ ] Define 10 test tasks across the EA workflow
- [ ] Run each, log results + token usage + time
- [ ] Compare against M2.7 baseline (where possible)
- [ ] Document findings in `learnings.md` + a permanent note
- [ ] Identify the 3 things M3 is dramatically better at
- [ ] Identify the 3 things M3 is no better (or worse) at

## Notes
*The eval is the work — every real interaction is a data point. Don't sandbag, don't cherry-pick.*
